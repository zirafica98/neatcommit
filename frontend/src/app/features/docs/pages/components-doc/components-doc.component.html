Components

The frontend is built with Angular standalone components. Below is a short description of each main feature component and where it is used.

On this page Public (no login) App (authenticated)

Public (no login)

LandingComponent

Path: features/landing/landing. Route: /.

Landing page: hero, features, pricing, how it works, documentation CTA, FAQ, footer. Entry point for new visitors.

NewsComponent

Path: features/news/news. Route: /news.

Displays a list of news/updates (date, title, body). Used for product news and roadmap.

DocsLayoutComponent

Path: features/docs/docs-layout. Route: /docs (layout).

Layout for the documentation section: sidebar navigation and router outlet for doc pages (overview, how-it-works, tech-stack, api, components).

Docs pages (Overview, How it works, Tech stack, API Reference, Components)

Path: features/docs/pages/*. Routes: /docs/overview, /docs/how-it-works, /docs/tech-stack, /docs/api, /docs/components.

Static content components that render the documentation content inside the docs layout.

LoginComponent

Path: features/auth/login. Route: /auth/login.

Login page: GitHub OAuth button, optional email/password login. Redirects after successful auth.

CallbackComponent

Path: features/auth/callback. Route: /auth/callback.

Handles OAuth callback: exchanges code for tokens, stores them, redirects to app or installation flow.

InstallComponent

Path: features/auth/install. Route: /auth/install.

Prompts user to install the GitHub App; links to GitHub App installation URL.

InstallationWaitComponent

Path: features/auth/installation-wait. Route: /auth/installation-wait.

Shown after redirect from GitHub App install; waits for webhook and then continues (e.g. redirect to app).

App (authenticated)

All of the following live under the main layout (MainLayoutComponent) and require authentication.

MainLayoutComponent

Path: layout/main-layout. Route: /app (wrapper).

Main app shell: sidebar/nav, outlet for dashboard, repositories, reviews, settings, documentation, search, help, pricing, admin.

HomeRedirectComponent

Path: layout/home-redirect. Route: /app (default child).

Redirects /app to a default route (e.g. dashboard).

DashboardComponent

Path: features/dashboard/dashboard. Route: /app/dashboard.

Dashboard: overview of repositories, recent reviews, security scores, quick actions.

RepositoriesComponent

Path: features/repositories/repositories. Route: /app/repositories.

Lists repositories connected via the GitHub App; manage which repos are analyzed.

ReviewsComponent

Path: features/reviews/reviews. Route: /app/reviews.

List of code reviews (PR analyses) with filters and links to detail.

ReviewDetailComponent

Path: features/reviews/review-detail. Route: /app/reviews/:id (or similar).

Shows a single review: findings, score, files, suggested fixes. Export options (e.g. PDF).

SettingsComponent

Path: features/settings/settings. Route: /app/settings.

User settings: profile, connected GitHub account, notifications, subscription link.

DocumentationComponent (in-app)

Path: features/documentation/documentation. Route: /app/documentation.

In-app feature for generating and downloading AI-generated documentation for a repository (different from the public /docs site).

SearchComponent

Path: features/search/search. Route: /app/search.

Search UI for reviews, issues, and repositories; calls GET /api/search.

HelpComponent

Path: features/help/help. Route: /app/help.

Help and support content, links to docs or contact.

PricingComponent

Path: features/pricing/pricing. Route: /app/pricing.

Pricing plans and upgrade flow; may use subscription API.

AdminComponent

Path: features/admin/admin. Route: /app/admin.

Admin panel: stats, user list, user detail. Protected by adminGuard (role ADMIN).
