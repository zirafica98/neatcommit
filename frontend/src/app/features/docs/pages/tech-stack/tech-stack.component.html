Tech stack

NeatCommit is built with a Node.js backend and an Angular frontend. This section summarizes the main technologies used.

Backend

Layer Technology
RuntimeNode.js 18+
LanguageTypeScript
FrameworkExpress.js
DatabasePostgreSQL 14+
ORMPrisma
Cache / QueueRedis 7+, BullMQ
AuthJWT (access + refresh tokens)
IntegrationsGitHub App (webhooks, REST API), OpenAI API
ParsingBabel (JS/TS AST), regex for other languages

Frontend

Layer Technology
FrameworkAngular 17+ (standalone components)
UIAngular Material
State / dataRxJS, Angular services
RoutingAngular Router (lazy-loaded features)

Backend structure (high level)

The backend is organized into api/routes (REST endpoints), services (business logic: auth, GitHub, analysis, LLM, security, subscription, documentation, export, search, admin), workers (BullMQ jobs for analysis and documentation), config (env, database, Redis, queue, Sentry), middleware (auth, validation, rate limiting, subscription checks), and utils (logger, AST parser, language detection, webhook verification).

Frontend structure (high level)

The Angular app uses core (guards, interceptors, layout), layout (main layout with nav), and features (landing, auth, dashboard, repositories, reviews, settings, documentation, search, help, pricing, admin, news). Routes are lazy-loaded per feature. The docs you are reading are a separate public section under /docs.
